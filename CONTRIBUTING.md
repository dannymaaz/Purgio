# Contribuyendo a Purgio

¡Gracias por tu interés en contribuir a Purgio! Este es un proyecto de código abierto diseñado para ser seguro, minimalista y ligero.

## Código de Conducta

Al participar en este proyecto, te comprometes a seguir nuestro [Código de Conducta](CODE_OF_CONDUCT.md) en todas tus interacciones.

## ¿Cómo puedo contribuir?

### Reportar Errores (Bugs)
Si encuentras un error, por favor utiliza la plantilla de reporte de errores y proporciona:
- Tu sistema operativo y versión.
- Pasos detallados para reproducir el problema.
- Comportamiento esperado y obtenido.
- Logs de consola o capturas relevantes.

### Sugerir Mejoras
Estamos abiertos a ideas para mejorar la UX/UI o el rendimiento del escaneo. Por favor, abre una discusión o un issue utilizando la plantilla de sugerencia de mejoras.

### Enviar Pull Requests (PRs)
1. Haz un fork del repositorio y crea tu rama a partir de `main` (ej: `feature/mi-mejora` o `bugfix/correccion-error`).
2. Asegúrate de que tu código siga las guías de estilo existentes.
3. El código de backend en Rust debe pasar todas las pruebas (`cargo test`) y lints (`cargo clippy`).
4. El frontend debe compilar sin errores de TypeScript (`npm run build`).
5. Escribe comentarios de código claros y concisos en español.
6. Abre un Pull Request describiendo detalladamente los cambios realizados.
